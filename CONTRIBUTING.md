# 贡献指南

感谢你对 HR 人力资源管理系统的关注与贡献。

## 提交前准备

1. Fork 仓库并从 `main` 分支创建新分支。
2. 本地安装依赖并确保项目可运行。
3. 提交前执行基础检查：
   - 后端：`python manage.py check`
   - 前端（web/mobile）：`npm run build` 或 `npm run lint`（如项目脚本可用）

## 分支与提交规范

1. 分支命名建议：`feature/<name>`、`fix/<name>`、`docs/<name>`。
2. 提交信息建议使用简洁祈使句，例如：`fix: correct attendance status mapping`。
3. 单个 PR 尽量聚焦一个主题，避免混入无关改动。

## Pull Request 要求

1. 说明变更背景、方案与影响范围。
2. 涉及 UI 的改动请附截图或录屏。
3. 涉及数据库模型变更请附迁移说明。
4. 若存在兼容性风险，请在描述中明确。

## 代码风格建议

1. 保持现有目录结构与命名风格。
2. 不要在同一 PR 中进行大规模无意义格式化。
3. 对复杂逻辑补充必要注释，避免冗余注释。

## 报告问题

欢迎通过 Issue 反馈 Bug 或提出改进建议。请尽量提供：

1. 复现步骤
2. 预期行为与实际行为
3. 运行环境（操作系统、Python/Node 版本、部署方式）
